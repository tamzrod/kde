# Deployment

Deployment contains stable implementations that have already been validated inside the Laboratory.

## Purpose

Deployment is where validated concepts become usable artifacts. These implementations have been tested in the Laboratory and are considered stable for use.

## Structure

```
deployment/
├── mcp/              # Production MCP implementations
├── cli/              # Production CLI implementations (future)
└── api/              # Production API implementations (future)
```

## Status

**Production-ready** - These implementations are validated and stable.

## Principles

- Deployment consumes validated knowledge
- Deployment does not drive methodology
- Only Laboratory-validated concepts reach Deployment
- Multiple implementations (MCP, CLI, API) evolve together with shared methodology

## Key Principle

> "Experiment before deployment."

Only concepts that have been validated in the Laboratory should be promoted to Deployment.
