# KDE Runtime: Knowledge-on-Demand + Skills

This directory contains the Knowledge-on-Demand Runtime implementation with dynamic skill loading.

## Components

### Knowledge-on-Demand (INV-019)

| Component | Description |
|-----------|-------------|
| `catalog.json` | Knowledge artifact index |
| `retrieval.py` | Retrieval engine |
| `sop005.py` | SOP-005 retrieval policy execution |
| `instrumentation.py` | Runtime logging and metrics |
| `attribution.py` | Decision origin tracking |
| `runtime.py` | Main runtime integration |

### Dynamic Skill Loading (INV-022)

| Component | Description |
|-----------|-------------|
| `skills/registry.json` | 8 promoted skills |
| `skills/loader.py` | Skill loader and resolver |
| `skills/__init__.py` | Skill module |

## Architecture

```
User Request
    │
    ▼
Runtime
    │
    ▼
Task Classification
    │
    ▼
Required Skills ──► Registry Lookup
    │
    ▼
Load Skills ──► Dependency Resolution
    │
    ▼
Retrieve Knowledge ──► Knowledge-on-Demand
    │
    ▼
Apply SOP
    │
    ▼
Build Context ──► Combined Skill + Knowledge Context
    │
    ▼
Execute Engine (unchanged)
    │
    ▼
Return Result
```

## Usage

### Knowledge-on-Demand

```python
from runtime.runtime import KnowledgeOnDemandRuntime

runtime = KnowledgeOnDemandRuntime('INV-022')
context = runtime.initialize(
    title='Continuation of Investigation',
    parent_id='INV-021',
    keywords=['architecture']
)
```

### Dynamic Skills

```python
from runtime.skills import SkillLoader

loader = SkillLoader()
skills = loader.select_skills_for_task(['continuation'])
loader.load_skills(skills, 'INV-022')
context_doc = loader.generate_context_document()
```
